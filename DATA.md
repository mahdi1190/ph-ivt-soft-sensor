# DATA.md — Dataset inventory & access

**Manuscript:** *Soft sensor based on pH for real-time monitoring of mRNA medicines production*  
**Authors:** Mahdi Ahmed, Shady Hamed, Ricardo Cardoso, Charley Kenyon, Manoj Pohare, Mabrouka Maamra, Mark Dickman, Joan Cordiner, Zoltan Kis  
**Corresponding author:** Zoltan Kis (z.kis@sheffield.ac.uk)

This document lists all datasets **used** and **generated** by the code in this repository, with short descriptions and how to recreate them.  
We include simple commands to recompute the generated artefacts from the provided scripts.

> **Licensing:** Unless otherwise stated, data distributed in this repository are under **CC BY 4.0**.  
> **Checksums:** If you wish to track integrity, compute SHA256 for each file locally (examples at the end).

---

## Conventions

- **Type**  
  - *processed*: curated/cleaned inputs committed to `data/` (used directly by scripts)  
  - *optional*: additional inputs used for baseline overlays (H–H)  
  - *generated*: written by scripts to `reports/` or `figures/`
- **Recreate**  
  Exact command(s) you can run from the repo root to reproduce the file(s).

## A) Processed inputs (committed)

| ID | Path | Type | Description | License | Recreate |
|----|------|------|-------------|---------|----------|
| D1 | `data/all_data_processed.xlsx` | processed | **Unified experimental workbook** with four sheets: `egfphepes`, `egfptris`, `csphepes`, `csptris`. Columns may include: `time, ph1, ph2, ph3, temp, rna1, rna2, rna3, qubit1, qubit2, ATP_tot, GTP_tot, CTP_tot, UTP_tot`. Scripts default to `rna2` for RNA and use per-sheet pH mapping: `egfphepes→ph2`, `egfptris→ph1`, `csphepes→ph2`, `csptris→ph3` (with fallback). | CC BY 4.0 | n/a (input) |

**Notes:**
- Missing values are expected (e.g., sparse RNA/timepoints); plotting scripts handle NaNs by dropping rows where the plotted variable is absent.
- Ensure sheet names exactly match: `egfphepes`, `egfptris`, `csphepes`, `csptris`.

## C) Model outputs (generated by running the soft-sensor)

> Produced by `code/soft_sensor_main.py`. These CSVs are consumed by the plotting scripts.

| ID | Path | Type | Description | License | Recreate |
|----|------|------|-------------|---------|----------|
| G1 | `reports/softsensor_run/ivt_ukf_results_egfp_HEPES.csv` | generated | UKF outputs for **eGFP + HEPES**: time series (`time_min`, `pH_pred`, `RNA`, species, ±2σ columns, etc.). | CC BY 4.0 | `python code/soft_sensor_main.py` |
| G2 | `reports/softsensor_run/ivt_ukf_results_egfp_TRIS.csv` | generated | UKF outputs for **eGFP + TRIS**. | CC BY 4.0 | `python code/soft_sensor_main.py` |
| G3 | `reports/softsensor_run/ivt_ukf_results_csp_HEPES.csv` | generated | UKF outputs for **CSP + HEPES**. | CC BY 4.0 | `python code/soft_sensor_main.py` |
| G4 | `reports/softsensor_run/ivt_ukf_results_csp_TRIS.csv` | generated | UKF outputs for **CSP + TRIS**. | CC BY 4.0 | `python code/soft_sensor_main.py` |

**What’s inside each UKF CSV (typical):**
- `time_min`, `pH_meas`, `pH_pred`, `pH_pred_minus_2σ`, `pH_pred_plus_2σ`
- `RNA` (g/L) ±2σ; NTP totals `ATP_tot`, `CTP_tot`, `GTP_tot`, `UTP_tot` (mM) ±2σ
- Selected hidden states and buffer pools (e.g., `HEP_base`, `H_HEP_acid`, `Acetate_base`, `Acetic_acid`, `Pi`, `PPi`, `MgATP/CTP/GTP/UTP`) with ±2σ.

## Recreate everything (at a glance)

From the repository root (with the conda environment activated):

```bash
# 1) Generate UKF outputs
python code/soft_sensor_main.py

# 2) RNA figure(s)
python code/plot_rna_quad.py

# 3) pH figure(s)
python code/plot_ph_quad.py

# 4) NTP figure(s)
python code/plot_ntp_quad.py
